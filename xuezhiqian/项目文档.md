## 6. API接口设计

### 6.1 基础配置

- **基础URL**: `https://api.xuezhiqian.com`（模拟地址）
- **请求超时**: 10000ms
- **请求头**: `Content-Type: application/x-www-form-urlencoded`

### 6.2 主要API接口

| 接口路径 | 方法 | 功能描述 | 参数 |
|---------|-----|---------|------|
| /auth/login | POST | 用户登录 | username, password, captcha |
| /auth/captcha | GET | 获取验证码 | 无 |
| /user/info | GET | 获取用户信息 | 无 |
| /concerts | GET | 获取演唱会列表 | page, pageSize, keyword |
| /albums | GET | 获取专辑列表 | page, pageSize, keyword |
| /newsongs | GET | 获取新歌预告 | page, pageSize, keyword |
| /:type/:id | GET | 获取详情信息 | id, type |
| /vip/info | GET | 获取VIP信息 | 无 |
| /vip/buy | POST | 购买VIP | plan |
| /vip/benefits | GET | 获取VIP权益 | 无 |

### 6.3 模拟数据

项目提供了完善的模拟数据支持，包含以下主要数据类型：

- 登录成功响应
- 演唱会数据
- 专辑数据
- 新歌预告数据
- VIP信息

## 7. 状态管理

使用Pinia进行状态管理，主要管理以下状态：

- **用户信息**：基本资料、登录状态、token
- **会员状态**：VIP等级、有效期
- **应援积分**：支持点数、谦币
- **应援等级**：等级、经验值、徽章
- **个人设置**：头像、背景、布局等
- **任务系统**：每日、每周、每月任务
- **收藏内容**：歌曲、专辑、新闻、活动
- **通知消息**：系统通知、互动消息
